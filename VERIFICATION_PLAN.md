## 2. 캐릭터 1종 표준 검증 절차

> **원칙: 먼저 스킬 전문을 다 읽고 "이렇게 작동해야 한다"를 추론한 뒤, 구현과 대조.**
> 무작정 돌려서 숫자만 보지 말고 *기대 동작 ↔ 실제 파싱*을 비교한다. 확실한 버그만 고치고,
> **불확실한 건 §비고에 모아 나중에 일괄 보고**.

1. **원문 읽기 + 기대동작 추론** — `python tools/verify.py <cid>` 로 평타/필살/룬필살/패시브1~5(+룬패시브)
   한글 설명을 먼저 통독하고, 각 스킬이 어떤 채널·타겟·조건으로 작동해야 하는지 머릿속에 모델을 세운다.
2. **파싱 점검** — ‼UNPARSED가 있으면 패턴을 §4 백로그에 분류 → 파서 leaf 추가.
3. **효과 의미 대조** — 추론한 기대동작 ↔ 효과테이블을 1:1 대조:
   - 채널(stat): 기초ATK% / ATK% / 고정 / 주는딜 / 평타뎀·EX효과·발동효과 / 받는딜 중 맞는가
   - 타겟(target): 자신 / 아군전체 / 역할·속성 필터 / 적 / 포지션 / 최저HP 맞는가
   - 조건(cond): 스택게이트·팀조건·while·확률 맞는가, 임계값(≥N) 맞는가
   - 지속(dur)·중첩(max_stacks): "N턴", "최대 N중첩" 반영됐나. **중첩 표기 없으면 단일·갱신**이 기본.
   - ⚠ *greedy 흡수* 주의: " and " 절이 안 잘려 뒤 효과가 사라지는 케이스(신리랑 반격 연환각) — UNPARSED도 안 뜸.
4. **트리거 버프 수신 검증** — `verify.py`가 자동으로 **제트블랙(10418) 동반 + 100%모드** 스모크 실행:
   - 제트블랙 = "보통공격 시 아군 전체 **발동효과 +24%**". 테스트 캐릭의 **발동 데미지 hit이 이 버프를 받는지**,
     그리고 **평타/필살 채널로 누수되지 않는지**(채널 분리)를 자동 점검 → "받아야 할 게 안 받음 / 안 받아야 할 게 받음" 색출.
5. **동작 테스트** — 단독 스모크 + 시너지 팀으로 대시보드 시뮬, **로그 드릴다운**으로 실제 수치 확인.
   엣지: 🎲100%모드, 중첩 상한, 조건 게이트 on/off, 2회행동(이태호류), 더블궁(임부언 피드).
6. **수정 → 재검증 → §6 매트릭스에 ✓**. 불확실분은 §비고 기록. 회귀 방지로 핵심 조합은 스냅샷 보관.
